# Configure MLflow Tracking Profiles with Express.js Policy API

You need to develop a Node.js CLI tool that queries an existing Express.js policy API and generates local MLflow tracking-server configurations based on the API's returned policies. 

## The API
The Express.js Policy API provides rules via the following endpoints (accessible at the `apiUrl` provided to the CLI):
- `GET /api/policies/:profileId` - Returns the policy for the given profile.
- `GET /api/port-assignments` - Returns allowed port ranges for each profile.
- `GET /api/filesystem-roots` - Returns allowed filesystem roots (for informational purposes; your CLI does not need to validate against this).

You can expect to encounter various profiles such as `dev-local` (typically assigned port `5000`), `prod-secure` (typically assigned port `5443`), and testing profiles like `test-max-retention`. Your tool must dynamically fetch and apply the correct values for whatever profile it is executed against.

## CLI Behavior
Your CLI tool (`/app/cli/index.js`) must accept three arguments:
- `-p, --profile <profileId>`
- `-d, --dir <targetDirectory>`
- `-u, --apiUrl <apiUrl>`

**Important**: The CLI file `/app/cli/index.js` must be directly executable by the test runner. Make sure you retain or apply the executable bit (`chmod +x`) if you rewrite the file.

When executed, the CLI must produce the following configuration in the `<targetDirectory>`:

1. **`mlflow-env.sh`**: An environment file exporting the variables specified in the policy's `environmentVars`, as well as `MLFLOW_TRACKING_URI`. The URI must be set to `sqlite://<targetDirectory>/mlflow.db` if the policy's `allowSqlite` is true. Otherwise, it must be `http://127.0.0.1:<assignedPort>`. Use strict double quotes for all export values (e.g., `export VAR="value"`).
2. **`mlflow-tracker.service`**: A systemd user unit file to start MLflow. It must use `/usr/local/bin/mlflow server` as the `ExecStart` command, configured with `--port <assignedPort>` and `--default-artifact-root <targetDirectory>/artifacts`. It must also specify `EnvironmentFile=<targetDirectory>/mlflow-env.sh`.
3. **`audit-manifest.json`**: A JSON file containing the keys `profileId`, `timestamp` (as a valid ISO 8601 string), `port`, and `retentionDays`.
4. **`artifacts` Directory**: An empty subdirectory named `artifacts` must be created inside the `<targetDirectory>`.

## Validation & Constraints
The output generated by the CLI must adhere to the following rules:
- **Port Assignment**: The assigned port from `/api/port-assignments` must be verified to exist within its defined `portRange`.
- **Retention**: The policy's `maxRetentionDays` must not exceed the system global maximum of 3650 days. If the API returns a policy exceeding this, the tool must cap it at 3650 days in the audit manifest.
- **Error Handling**: The tool must exit gracefully with a non-zero status code and an error message if the API returns a 404 for the profile or if the assigned port is out of range.

The Express API is pre-configured and the CLI structure is ready for you in `/app/cli/index.js`.
